Web Page Files

A separate HTML page is created for each published vacancy. Two vacancy index files are also created, these contain links to the published vacancy pages listed in publication date order. Occupation indexes and occupation sub-indexes can be optionally generated so that your published vacancies can be viewed by occupation.

 

The default PRS behaviour is to put permanent vacancies in the first index (the pi index) and to put contract vacancies in the second index (the ci index) but the PRS allows for customised Web Index Categorisation.

 

The published HTML vacancy and index pages are named are named as follows:

 

File Name

Description

<vid>.html

Vacancy file, <vid> is the PRS Vacancy ID formatted as an 8 digit number with leading zeros.

ci.html

The contract vacancies index.

pi.html

The permanent vacancies index.

oi.html

The occupations index.

oi_<id>.html

An occupation sub-index, <id> is a four digit number identifying the occupation.

 

 

NOTES:

1.By default, exported Web page files are written to the WebPages sub-directory of the PRS VacancyFiles Export folder.  You can change Export Folder location by setting the PRS Export Folder option.
2.All file names will be prefixed with the Company ID followed by an underscore if the PRS Company ID has been set (see the PRS Tools->Options... menu item).
3.All files with a .html file extension are deleted from the Web page export folder before HTML page generation begins to ensure the Web page folder only contains current published vacancy pages.
4.The occupations index file and occupations sub-index files will only be generated if the Generate occupation indexes option in the Vacancy Publication Options dialog is checked (Tools->Vacancy Publication->Publication Options).
