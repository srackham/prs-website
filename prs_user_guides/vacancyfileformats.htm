Vacancy File Formats

In addition to exporting to HTML Web pages, published vacancies can be exported to Microsoft Excel, CSV (Comma Separated Values) and XML files.

 

Supported vacancy file formats:

 

Format

Exported File Name

CSV

v<cid><mmdd>.csv

Excel

v<cid><mmdd>.xls

XML

v<cid><mmdd>.xml

 

NOTES:

1.By default, exported vacancy files are written to a sub-directory of the PRS VacancyFiles Export folder.  You can change Export Folder location by setting the PRS Export Folder option.
2.<cid> is the optional Company ID global option.
3.<mmdd> is the publication date expressed as a two digit month number followed by a two digit day of month number (omitted from file name if the Include date in file names vacancy files option is unchecked (see below)).
4.If the Include date in file names vacancy files option is unchecked the <mmdd> date is omitted from the vacancy file name. Press the Vacancy File Options... button on the Vacancy Publication Tool form access this option.
5.CSV and Excel file formats can be fully customised. Press the Vacancy File Options... button on the Vacancy Publication Tool form and then press the Advanced... button.
6.The XML file format can be fully customised. Simply edit the XML header, footer and vacancy templates (v_tpl.xml, v_footer_tpl.xml, v_tpl.xml) located in the PRS Shared Data folder.

 

For example, if the Export Folder is set to \\QUARK\public\Export\ then vacancy file publication on the 25th of May from company with ID 'FB' would export published vacancies to an Excel formatted file named \\QUARK\public\Export\VacancyFiles\VFB0525.XLS.
